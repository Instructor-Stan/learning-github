Cloning a repository using gitbash.

  • Change to a high level directory where your git projects with be stored (ie. "cd" - with no paramter, puts you in your home directory.
  • Type "git init" to setup the files Git will need to manage files on your PC.
  • Type "git clone git@github.com:StanCrump/learning-github.git" to clone this repository. The clone command can be found on the web under the "Code" button in the repository you want to clone.
  • Type "cd <directory> where directory is the part of the name you cloned between the last slash (/) and ".git" on the end. Ex. learning-github
  • Type "git status" to confirm git knows about your files.
  • Type "code ." to start VS Code editor and open the directory you just cloned. (Now change some code. I created a new file "index.html")
  • Type "git status" to see that git has acknowledged your changes and git prompts you to the next step(s).
  • Type "git add index.html" to add a new file "index.html" to the repository.
  • Type "git commit -m "Your comments describing all your changes so the approver will know what changes to look for."
  • Type "git push origin main" to send your changes to the github web repository. The origin was set to "origin" with the "git clone" command and since we did not create a branch, you will send your changes to the "main" branch. Your code changes should now be viewable on github (the web tool.)

Updating github with your changes.

  • Type "git status" to see that git has acknowledged your changes and git prompts you to the next step(s).
  • Type "git add index.html" to add a new file "index.html" to the repository. If you have no new files, you can skip this step.
  • Type "git commit -m "Your comments describing all your changes so the approver will know what changes to look for."
  • Type "git push origin main" to send your changes to the github web repository. The origin was set to "origin" with the "git clone" command and since we did not create a branch, you will send your changes to the "main" branch. Your code changes should now be viewable on github (the web tool.)
